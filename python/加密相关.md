# 加密服务

## hashlib--安全散列与消息摘要

该模块实现了许多不同安全等级的散列和消息摘要算法的API。包括FIPS安全散列算法SHA1，SHA224，SHA256，SHA384和SHA512以及RSA的MD5算法。术语“安全散列”和“消息摘要”是可互换的。较旧的算法称为消息摘要，现代术语是安全散列。注：如果想找adler32或crc32哈希函数，它们在zlib 模块中。

常用的属性和方法：

+ hashlib.algorithms_available：包含正在运行的Python解释器中可用的哈希算法名称的集合。
+ hashlib.algorithms_guaranteed：包含在所有平台上保证由此模块支持的哈希算法名称的集合。
+ hash.digest_size：散列函数产生的字节大小
+ hash.block_size：散列算法的内部块大小
+ hash.name：哈希的规范名称，始终为小写，并始终适合作为new()创建此类型的另一个哈希的参数
+ md5()/sha1()：创建一个md5或者sha1加密模式的hash对象
+ hash.update(arg)：使用类似字节的对象更新哈希对象。重复调用相当于单个调用，并且所有参数都连接在一起：：m.update(a); m.update(b)，则等于m.update(a+b)。
+ hash.digest()：返回二进制数据字符串值摘要
+ hash.hexdigest()：返回十六进制数据字符串值摘要。这可用于在电子邮件或其他非二进制环境中安全地交换值。
+ hash.copy()：返回哈希对象的副本。这可以用于有效地计算共享公共初始子字符串的数据的摘要。

## getpass--便携式密码输入工具

提供了两个函数：

+ getpass.getpass(prompt='Password: ', stream=None)：提示用户输入一个密码且不会回显。用户会看到prompt字符串作为提示，其默认值为'Password: '。在Unix上，如有必要提示会使用替换错误句柄写入到文件类对象stream。stream默认指向控制终端(/dev/tty)，如果不可用则指向sys.stderr(此参数在Windows上会被忽略)。如果getpass()回显输入不可用则将回退为打印一条警告消息到stream并且从sys.stdin读取时发出GetPassWarning。
+ getpass.getuser()：返回用户的登录名称。此函数会按顺序检查环境变量LOGNAME,USER,LNAME和USERNAME，并返回其中第一个被设置为非空字符串的值。如果均未设置，则在支持pwd模块的系统上将返回来自密码数据库的登录名，否则将引发一个异常。通常情况下，此函数应优先于os.getlogin()使用。
