# 网络相关

## urllib--URL处理模块

urllib 是一个收集了多个使用URL的模块的软件包：

+ urllib.request 打开和请求 URLs。更高层级的HTTP客户端API可使用 Requests包。
+ urllib.error 包含 urllib.request 抛出的异常
+ urllib.parse 用于处理 URL
+ urllib.robotparser 用于解析 robots.txt 文件

## http--HTTP模块

http是一个包，它收集了多个用于处理超文本传输协议的模块:

+ http.client 是一个低层级的HTTP协议客户端；对于高层级的URL访问请使用urllib.request
+ http.server 包含基于 socketserver 的基本 HTTP 服务类
+ http.cookies 包含一些有用来实现通过 cookies 进行状态管理的工具
+ http.cookiejar 提供了 cookies 的持久化

## uuid--根据RFC 4122生成UUID对象

该模块提供不可变的UUID对象（UUID类）和函数uuid1()，uuid3()，uuid4()，uuid5()。如果想要的只是一个唯一的ID，应该调用uuid1()或uuid4()。请注意，uuid1()可能会破坏隐私，因为它会创建包含计算机网络地址的UUID。而uuid4()创建一个随机UUID。根据底层平台的支持，uuid1()可能会也可能不会返回“安全”UUID。安全UUID是使用同步方法生成的UUID，确保没有两个进程可以获得相同的UUID。所有UUID具有is_safe属性的实例都使用此枚举来中继有关UUID安全性的任何信息。

## xmlrpc--XMLRPC 服务端与客户端模块

XML-RPC 是一种远程过程调用方法，它使用通过 HTTP 传递的 XML 作为载体。有了它，客户端可以在远程服务器上调用带参数的方法（服务器以URI命名）并获取结构化的数据。

xmlrpc 是一个集合了 XML-RPC 服务端与客户端实现模块的包。 这些模块是：

+ xmlrpc.client：支持编写XML-RPC客户端代码; 它处理在一致的Python对象和XML之间进行转换的所有细节。
+ xmlrpc.server：为使用Python编写的XML-RPC服务器提供基本服务器框架。服务器可以是独立的，使用SimpleXMLRPCServer，也可以嵌入在CGI环境中使用 CGIXMLRPCRequestHandler。
